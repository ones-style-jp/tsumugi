# Tsumugi マルチテナント本番セットアップ手順

> 法人ごとに Supabase プロジェクト、法人内の複数店舗はマルチテナント方式

## 全体の流れ

1. SQL の追加実行 (10_multi_tenant.sql)
2. 既存テストデータの初期化 (各端末)
3. 本部管理者でログイン → 動作確認
4. 2 店舗目を本部から追加
5. 各店舗のスタッフアカウントを発行
6. 各店舗で運用開始

---

## STEP 1: 追加 SQL の実行

1. https://supabase.com/dashboard → `tsumugi-prod` プロジェクトを開く
2. 左サイドバー「**SQL Editor**」 → 「New query」
3. リポジトリの [`supabase/10_multi_tenant.sql`](./10_multi_tenant.sql) の中身を全コピー → 貼り付け → 「**Run**」
4. 結果に以下が出れば成功:
   - `stores | 1` (扇橋店が初期作成)
   - `staff  | 2` (本部管理者 + 扇橋店管理者)
   - `app_state | ...`

---

## STEP 2: 各端末のテストデータを初期化

各端末 (PC、iPad、スマホ) で以下を実行:

1. Chrome の場合: `F12` キー → 「Console」タブ
2. Safari の場合: 設定 → 詳細 → 開発メニューを表示 → 開発メニュー → JavaScript コンソール
3. 以下をコピー → ペースト → Enter:

```js
localStorage.clear();
sessionStorage.clear();
location.reload();
```

これでブラウザ内の古いテストデータが消えます。

---

## STEP 3: 本部管理者でログイン

1. 本番 URL `https://tsumugi.ones-style.co.jp` を開く
2. **「事業所スタッフ専用ログイン」** 画面が表示される
3. 以下でログイン:

| 項目 | 値 |
|------|-----|
| ID | `admin` |
| パスワード | `tsumugi2026!` |

⚠️ **ログイン後すぐにパスワードを変更してください** (本部管理者の権限が漏れると全店舗のデータが見える)

4. ログイン成功 → 「本部管理コンソール」が表示される
5. 「ひかりデイサービス扇橋店」が表示されているはず

---

## STEP 4: 2 店舗目を追加

1. 本部管理コンソール画面 → 「**+ 店舗を追加**」
2. 以下を入力:

| 項目 | 例 |
|------|-----|
| 店舗ID | `store_honjo` (半角英数字、後から変更困難) |
| 店舗名 (正式) | `ひかりデイサービス本所店` |
| 短縮名 | `本所店` |
| 法人名 | `ワンズスタイル株式会社` |
| 電話番号 | `03-XXXX-XXXX` |

3. 「店舗を作成」

---

## STEP 5: 各店舗のスタッフアカウントを発行

1. 本部管理コンソール画面 → 「**+ スタッフを追加**」
2. 以下を入力:

| 項目 | 例 (扇橋店管理者) |
|------|------------|
| 店舗 | ひかりデイサービス扇橋店 |
| 姓・名 | 田中・太郎 |
| 役割 | 店舗管理者 |
| ログインID | `ougibashi_tanaka` |
| 初期パスワード | (8文字以上の文字列、メモして店舗に伝える) |
| メール | `tanaka@example.com` |

3. 「スタッフを作成」

4. 各店舗ぶん繰り返す

---

## STEP 6: 動作確認

1. 本部管理コンソールの「ひかりデイサービス扇橋店」横の「**この店舗を開く →**」を押す
2. 通常の Tsumugi 画面 (利用者一覧など) に切り替わる
3. 試しに利用者を 1 人登録
4. ブラウザを閉じて新しいタブで本番 URL を開く
5. 今度は **扇橋店スタッフのアカウント** でログインする
   - 通常の Tsumugi 画面が直接開く (扇橋店専用)
   - 先ほど登録した利用者が見える ✅
6. 別ブラウザ (シークレットモード) で本所店のアカウントでログイン
   - 扇橋店の利用者は **見えない** ✅
   - これでデータ分離が確認できる

---

## 注意点

### 本部管理者の権限
- 全店舗のデータが見える + 編集できる
- 店舗追加/削除/スタッフ発行が可能
- → **本部 ID/PW は厳重に管理してください**

### 店舗管理者の権限
- 自店舗のデータのみ全権
- 自店舗のスタッフを追加できる (今は本部経由のみ、後日追加予定)

### 店舗スタッフの権限
- 自店舗のデータのみ閲覧・記録入力可能
- 利用者削除や設定変更は不可 (今後実装)

### パスワード変更
- 各スタッフは「設定」画面からパスワード変更可能 (実装予定)
- 緊急の場合は本部管理者が新パスワードを発行可能

---

## トラブルシューティング

### ログインしても画面が真っ白
- ブラウザを再読込 (Cmd+R / F5)
- それでもダメなら DevTools コンソールでエラー確認 → スクショで連絡

### 「IDまたはパスワードが違います」
- 大文字小文字の違いに注意
- 半角/全角の確認
- パスワードを忘れた場合は本部から再発行

### 「Supabase 未接続」
- Vercel の環境変数が反映されていない
- Vercel → Deployments → 最新を Redeploy
